# EasyCAT-STM32F3-Servo-Interface-Demo

EtherCAT master demo for Linux using IgH, controlling a servo via an STM32 Nucleo-F303RE with an EasyCAT (LAN9252) shield running SOES.

**Data flow:** Linux (IgH master) → Ethernet → EasyCAT/LAN9252 (ESC) ↔ SPI ↔ STM32F303RE (SOES slave) → PWM → Servo

## Hardware Required

- STM32 Nucleo-F303RE + EasyCAT shield (`spi_on_13_12_11` firmware variant)
- USB-C to Ethernet adapter (for the Linux PC — STM also connected via USB for flashing)
- Short RJ45 patch cables
- Servo: MG 996R (6V, 2.5A) or any 50Hz PWM servo
- Power supply for the servo (separate from the Nucleo)

## Quick Start

### 1. Install IgH EtherCAT Master

```bash
sudo apt update
sudo apt install ethercat-master ethercat-dkms libethercat-dev
```

### 2. Configure the Master

Find your Ethernet adapter:

```bash
ip link
```

Edit `/etc/ethercat.conf`:

```
MASTER0_DEVICE="aa:bb:cc:dd:ee:ff"   # replace with your adapter's MAC
DEVICE_MODULES="generic"
```

Apply the config:

```bash
sudo systemctl restart ethercat
sleep 2
sudo ethercat master      # should show "Master 0: Operational"
sudo ethercat slaves      # should show your EasyCAT slave
```

> **Network Manager conflict?** If NetworkManager keeps trying to manage the adapter, set it as unmanaged:
> ```bash
> sudo nmcli device set <interface> managed no
> ```

### 3. Flash the STM32 Firmware

Open `soes-stm32f3/` in STM32CubeIDE, build, and flash the target.  
The firmware uses the SOES EtherCAT slave stack with a custom object dictionary (`objectlist.c`, `utypes.h`) that maps:

- **RxPDO** (ESC register `0x0005`): receives a servo angle byte (0–180°) from the master
- **TxPDO** (ESC register `0x0006`): sends back debug/status bytes

The servo PWM is generated by TIM2 on pin PA0 at 50 Hz. The callback `cb_set_outputs()` converts the received angle to a pulse width (500–2500 µs).

### 4. Build & Run the Master App

```bash
gcc -o master_app master_app.c -lethercat
sudo ./master_app
```

The app starts a control loop at ≈1 kHz that sweeps a servo angle (0° → 180° → 0°) and prints debug output.

### 5. Wiring

| Component         | Connection                |
|-------------------|---------------------------|
| Linux PC          | USB-C Ethernet → EasyCAT RJ45 IN |
| STM32 Nucleo      | USB to PC (for flashing/debug)   |
| EasyCAT           | J5 (SPI header) → STM32 CN7      |
| Servo signal      | PA0 (Nucleo pin A0)             |
| Servo power       | External 6V PSU                 |
| Servo ground      | Common GND with Nucleo          |

## How It Works
There is no XML parsing. For this demo, all is hardcoded in C, as industrial software like TwinCAT were not integrated:

| Layer | File | Role |
|-------|------|------|
| **Linux master** | `master_app.c` | IgH `ecrt.h` API — requests master, registers PDO entries by vendor/product code, runs 1 kHz control loop |
| **Slave stack** | `lib/soes/` | SOES (Simple Open EtherCAT Slave) — handles ESC communication, CoE, PDO mapping on the STM32 |
| **Object dictionary** | `objectlist.c` + `utypes.h` | Defines the CANopen object dictionary entries and their memory layout — this is where the servo angle byte is mapped to `led[0].state` |
| **Firmware logic** | `Core/Src/main.c` | `cb_set_outputs()` reads the received angle and drives TIM2 PWM on PA0 |

The `EasyCAT.xml` file in the root is an **ESI (EtherCAT Slave Information) file** — purely documentation. It describes the same PDO mapping in a standard XML format for use with other EtherCAT masters (e.g. Beckhoff TwinCAT). Neither IgH nor SOES reads it at runtime.

## Project Structure

```
├── master_app.c              # IgH master app (runs on Linux) — all the master logic
├── master_app                # Pre-compiled binary
├── EasyCAT.xml               # ESI file (optional — documentation only)
├── soes-stm32f3/
│   ├── Core/Src/main.c       # Slave firmware entry — PWM servo control
│   ├── lib/soes/             # SOES EtherCAT slave stack (C library)
│   ├── lib/soes-esi/
│   │   ├── objectlist.c      # CANopen object dictionary (C code)
│   │   └── utypes.h          # Custom data types for the OD
│   └── twincat/              # TwinCAT project files (optional)
└── pdserv.sh                 # (optional) PDO monitoring script
```

## Troubleshooting

| Problem                          | Likely Fix                                    |
|----------------------------------|-----------------------------------------------|
| `ethercat master` shows no device | Wrong MAC in `/etc/ethercat.conf`            |
| `ethercat slaves` shows empty     | NetworkManager interference; `nmcli` unmanage |
| `Failed to register PDO entries`  | Vendor/product code mismatch in `master_app.c` |
| Servo not moving                  | Check external PSU, common GND, PA0 wiring    |
| STM32 not detected over USB       | Install ST-Link drivers or check USB cable    |

When unplugging the EasyCAT, first safely eject the virtual USB drive and stop the master:

```bash
sudo systemctl stop ethercat
```
